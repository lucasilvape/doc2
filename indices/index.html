    Status dos Índices por Produto¶

    Esta página apresenta o status dos índices implementados em cada produto dos pipelines MongoDB.

    Por que Índices são Importantes?¶

    Os índices em bancos de dados são estruturas auxiliares que aceleram drasticamente as consultas e melhoram a performance geral do sistema.

    Benefícios dos Índices:¶

    Performance de Consultas¶

    • Busca mais rápida: Reduzem o tempo de consulta de minutos para milissegundos
    • Menos I/O: Evitam varreduras completas da tabela (full table scan)
    • Escalabilidade: Mantêm performance mesmo com milhões de registros

    Operações Otimizadas¶

    • WHERE clauses: Filtros por uuid_bilhete, data_pagamento, etc. ficam instantâneos
    • ORDER BY: Ordenação por campos indexados é muito mais rápida
    • JOIN: Junções entre tabelas usando campos indexados são otimizadas
    • Agregações: COUNT, SUM, GROUP BY se beneficiam dos índices

    Casos de Uso Específicos nos Pipelines¶

    Exemplos Práticos

    Sem Índice: 

    -- Busca pode levar 30+ segundos em tabelas grandes
SELECT * FROM innoveo_picpay_auto 
WHERE uuid_bilhete = '12345-67890-abcd';

    Com Índice: 

    -- Mesma busca em < 10ms
SELECT * FROM innoveo_willbank_carteira 
WHERE uuid_bilhete = '12345-67890-abcd';

    Impacto nos Relatórios¶

    • Dashboards: Consultas de BI ficam muito mais responsivas
    • Relatórios periódicos: Processamento de grandes volumes em menos tempo
    • Análises ad-hoc: Exploração interativa de dados sem travamentos

    Legenda¶

    • ✅ Possui Índices: O produto tem índices configurados
    • ❌ Sem Índices: O produto não possui índices configurados

    Detalhes dos Índices por Produto¶

    Disney AP¶

    Tabela: innoveo_disney_ap

    • idx_innoveo_disney_ap_uuid_bilhete → uuid_bilhete
    • idx_innoveo_disney_ap_bilhete → info_numeroproposta
    • idx_innoveo_disney_ap_data_pagamento → data_pagamento
    • idx_innoveo_disney_ap_data_vencimento → data_vencimento

    Metlife Celular¶

    Tabela: innoveo_metlife_celular

    • idx_innoveo_metlife_celular_uuid_bilhete → uuid_bilhete
    • idx_innoveo_metlife_celular_bilhete → info_numeroproposta

    PicPay Cyber¶

    Este produto possui duas tabelas distintas com índices específicos:

    Tabela de Informações (innoveo_picpay_cyber_infos): - idx_innoveo_picpay_cyber_infos_uuid_bilhete → uuid_bilhete - idx_innoveo_picpay_cyber_infos_bilhete → info_quotecounter

    Tabela de Parcelas (innoveo_picpay_cyber_parcelas): - idx_innoveo_picpay_cyber_parcelas_uuid_bilhete → uuid_bilhete - idx_innoveo_picpay_cyber_parcelas_uuidinstallment → uuidinstallment - idx_innoveo_picpay_cyber_parcelas_certificate → certificate - idx_innoveo_picpay_cyber_parcelas_data_pagamento → data_pagamento - idx_innoveo_picpay_cyber_parcelas_data_vencimento → data_vencimento

    Análise Estatística dos Índices¶

    Colunas Mais Indexadas¶

    As colunas que aparecem com mais frequência nos índices são:

    1. uuid_bilhete: 7 produtos (100% dos produtos com índices)
    2. data_pagamento: 4 produtos (Disney AP, PicPay Cyber Parcelas, Telefonica Cyber, Willbank Carteira, XP Card)
    3. data_vencimento: 4 produtos (Disney AP, PicPay Cyber Parcelas, Telefonica Cyber, XP Card)
    4. info_quotecounter: 4 produtos (PicPay Cyber Infos, PicPay Phone, Willbank Carteira, XP Card)
    5. info_numeroproposta: 3 produtos (Disney AP, Metlife Celular, Telefonica Cyber)
    6. uuidinstallment: 3 produtos (PicPay Cyber Parcelas, PicPay Phone, Willbank Carteira)

    Produtos com Mais Índices¶

    1. Willbank Carteira: 9 índices (mais completo)
    2. PicPay Cyber: 7 índices (2 tabelas combinadas)
    3. Disney AP, Telefonica Cyber, XP Card: 4 índices cada
    4. PicPay Phone: 3 índices
    5. Metlife Celular: 2 índices (mais simples)

    Implementação Gradual

    Recomenda-se implementar os índices de forma gradual, começando pelos produtos de maior volume e maior frequência de consultas.
