    Ferramentas do Projeto¶

    O diretório tools/ contém módulos utilitários reutilizáveis que padronizam operações comuns entre todos os pipelines. Essas ferramentas abstraem complexidades de conectividade, transformações de dados e operações de banco, garantindo consistência e manutenibilidade.

    1. Conexão MongoDB (conexao_mongo.py)¶

    Gerencia conexões com clusters MongoDB de produção e homologação com autenticação segura.

    abrir_conexao_mongo(ambiente)¶

    Estabelece conexão com o cluster MongoDB baseado no ambiente especificado.

    Funcionalidades:

    • Ambientes separados: Produção e Homologação com credenciais distintas

    • Autenticação segura: Carregamento de credenciais via .env

    • Clusters específicos: URIs otimizadas para cada ambiente

    • Configurações otimizadas: TLS para produção, preferência analítica para homologação

    Source code in tools/conexao_mongo.py

Note:

    - Credenciais devem estar configuradas no arquivo .env

    def abrir_conexao_postgres(ambiente:Literal['PRODUCAO','HOMOLOGACAO']) -> psycopg2:
"""
Estabelece conexão com o banco PostgreSQL baseado no ambiente especificado.

Esta função cria uma conexão segura com o PostgreSQL utilizando credenciais
específicas do ambiente (produção ou homologação) carregadas de variáveis
de ambiente configuradas no arquivo .env.

Args:
    ambiente (Literal['PRODUCAO','HOMOLOGACAO']): Ambiente de destino para
        a conexão. Define qual conjunto de credenciais será utilizado.

Returns:
    psycopg2.connection: Objeto de conexão PostgreSQL configurado e ativo.

Raises:
    Exception: Se o ambiente especificado não for 'PRODUCAO' ou 'HOMOLOGACAO'
    ValueError: Se houver falha na conexão (credenciais inválidas, host inacessível, etc.)

Examples:
    >>> conn = abrir_conexao_postgres('PRODUCAO')
    >>> cur = conn.cursor()
    >>> cur.execute("SELECT 1")
    >>> conn.close()

    >>> conn = abrir_conexao_postgres('HOMOLOGACAO')
    >>> # Usar conexão para operações de teste
    >>> conn.close()

Note:
    - Variáveis de ambiente necessárias:
      - PRODUCAO: PG_HOST, PG_PORT, PG_DB, PG_USER, PG_PASSWORD
      - HOMOLOGACAO: PG_HOMOLOG_HOST, PG_HOMOLOG_PORT, etc.
    - Sempre feche a conexão após o uso
    - A conexão é estabelecida via string de conexão PostgreSQL padrão
"""

if ambiente == 'PRODUCAO':
    host = os.getenv("PG_HOST")
    port = os.getenv("PG_PORT")
    db   = os.getenv("PG_DB")
    user = os.getenv("PG_USER")
    pwd  = os.getenv("PG_PASSWORD")
elif ambiente == 'HOMOLOGACAO':
    host = os.getenv("PG_HOMOLOG_HOST")
    port = os.getenv("PG_HOMOLOG_PORT")
    db   = os.getenv("PG_HOMOLOG_DB")
    user = os.getenv("PG_HOMOLOG_USER")
    pwd  = os.getenv("PG_HOMOLOG_PASSWORD")
else:
    raise Exception('Selecione um dos ambientes : PRODUCAO ou HOMOLOGACAO')

url_format =  f"postgresql://{user}:{pwd}@{host}:{port}/{db}"

try:
    con = psycopg2.connect(url_format)
except Exception as e:
    print(f'###########  Erro ao efetuar a conexão cnx_str: {url_format}')
    raise ValueError(e)
return con

    def importar_csv_copy_alinhado(ambiente: str, caminho_csv: str, nome_tabela: str, colunas_tabela: list, separador='|'):
"""
Importa dados CSV para PostgreSQL usando o comando nativo COPY.

Esta função utiliza o comando COPY do PostgreSQL para importação otimizada
de grandes volumes de dados, oferecendo performance superior ao INSERT
tradicional.

Args:
    ambiente (str): Ambiente do banco ('PRODUCAO' ou 'HOMOLOGACAO')
    caminho_csv (str): Caminho absoluto para o arquivo CSV
    nome_tabela (str): Nome da tabela de destino
    colunas_tabela (list[str]): Lista ordenada das colunas da tabela
    separador (str, optional): Separador de campos do CSV. Defaults to '|'.

Raises:
    ValueError: Se houver erro no processamento do arquivo CSV

Examples:
    >>> colunas = ['uuid_bilhete', 'valor_parcela', 'data_criacao']
    >>> importar_csv_copy_alinhado(
    ...     'PRODUCAO', 
    ...     '/path/to/data.csv', 
    ...     'innoveo_kovr_ap', 
    ...     colunas
    ... )

Note:
    - Utiliza COPY com FORMAT CSV para máxima compatibilidade
    - Configurado para tratar NULL como string vazia
    - HEADER TRUE assume primeira linha como cabeçalho
    - Escape de caracteres especiais configurado automaticamente
"""
conn = abrir_conexao_postgres(ambiente=ambiente)
cur = conn.cursor()
try:
    comando = sql.SQL("""
    COPY {tabela} ({colunas})
    FROM {arquivo}
    WITH (
        FORMAT CSV,
        DELIMITER {separador_coluna},
        NULL '',
        HEADER TRUE,
        QUOTE '"',
        ESCAPE '\\'
    )
    """).format(
        tabela=sql.Identifier(nome_tabela),
        colunas=sql.SQL(', ').join(map(sql.Identifier, colunas_tabela)),
        separador_coluna=sql.Literal(separador),
        arquivo=sql.Literal(caminho_csv)
    )

    cur.execute(comando)
    conn.commit()
    cur.close()
    conn.close()
except Exception as e:
    if cur:
        cur.close()
    if conn:
        conn.close()
    print(caminho_csv)

    raise ValueError(f'Erro no processamento do arquivo: {caminho_csv} Erro: {e}')

    def remover_indices_postgres(ambiente: Literal['PRODUCAO','HOMOLOGACAO'], nm_tabela: str):
"""
Remove todos os índices de uma tabela, preservando constraints (PK/FK/UNIQUE).

Esta função otimiza cargas de dados removendo índices não-essenciais
antes de operações de importação em massa, melhorando significativamente
a performance de inserção.

Args:
    ambiente (Literal['PRODUCAO','HOMOLOGACAO']): Ambiente do banco
    nm_tabela (str): Nome da tabela para remover índices

Examples:
    >>> remover_indices_postgres('PRODUCAO', 'innoveo_kovr_ap')
    # Remove índices como idx_kovr_ap_uuid_bilhete, mas preserva PKs

Note:
    - Preserva índices de PRIMARY KEY, FOREIGN KEY e UNIQUE constraints
    - Remove apenas índices criados manualmente para otimização
    - Operação segura - não afeta integridade referencial
    - Use antes de cargas grandes, recrie índices após importação
    - Schema padrão é 'public', ajuste se necessário
"""
"""
Remove todos os índices de uma tabela, exceto índices de PK/FK/unique constraints.
"""
schema = 'public'  # ajuste se necessário
conn = abrir_conexao_postgres(ambiente=ambiente)

try:
    with conn.cursor() as cur:
        # Buscar índices que não são constraints
        cur.execute("""
            SELECT indexname
            FROM pg_indexes
            WHERE schemaname = %s
                AND tablename = %s
                AND indexname NOT IN (
                    SELECT conname
                    FROM pg_constraint
                    WHERE conrelid = %s::regclass
                )
        """, (schema, nm_tabela, nm_tabela))

        indexes = cur.fetchall()

        for idx_name, in indexes:
            print(f"Dropping index {idx_name}...")
            cur.execute(sql.SQL("DROP INDEX IF EXISTS {}").format(sql.Identifier(idx_name)))

    conn.commit()
    print("Todos os índices removidos (exceto constraints).")

finally:
    conn.close()

    def criar_indices(ambiente: Literal['PRODUCAO','HOMOLOGACAO'],indexes: dict, nm_tabela: str):
"""
Cria índices não-únicos em uma tabela PostgreSQL de forma segura.

Esta função cria índices de otimização para melhorar performance de queries,
utilizando CREATE INDEX IF NOT EXISTS para evitar erros em execuções
repetidas.

Args:
    ambiente (Literal['PRODUCAO','HOMOLOGACAO']): Ambiente do banco
    indexes (dict): Dicionário {nome_indice: coluna_ou_lista_colunas}
    nm_tabela (str): Nome da tabela onde criar os índices

Examples:
    >>> indices = {
    ...     'idx_kovr_ap_uuid_bilhete': 'uuid_bilhete',
    ...     'idx_kovr_ap_data_criacao': 'data_criacao',
    ...     'idx_kovr_ap_composto': ['uuid_bilhete', 'data_criacao']
    ... }
    >>> criar_indices('PRODUCAO', indices, 'innoveo_kovr_ap')

Note:
    - Sempre cria índices não-únicos para evitar conflitos
    - Suporta índices simples (uma coluna) e compostos (múltiplas colunas)
    - IF NOT EXISTS evita erros se índice já existir
    - Use após cargas de dados para otimizar consultas
    - Operação com commit automático
"""
"""
Cria índices não-únicos na tabela se não existirem.

:param indexes: dicionário {nome_do_indice: coluna OU lista de colunas}
"""
conn = abrir_conexao_postgres(ambiente=ambiente)
print(f'{fc.data_hora()} Criando índices...')
try:
    with conn.cursor() as cur:
        for index_name, columns in indexes.items():
            # Garante que columns seja lista
            if isinstance(columns, str):
                columns = [columns]

            cols_sql = sql.SQL(", ").join(map(sql.Identifier, columns))
            # REMOVE dynamismo do UNIQUE, sempre non-unique
            unique_clause = sql.SQL("")  # sempre vazio

            query = sql.SQL("CREATE {}INDEX IF NOT EXISTS {} ON {} ({})").format(
                unique_clause,
                sql.Identifier(index_name),
                sql.Identifier(nm_tabela),
                cols_sql
            )

            cur.execute(query)
            print(f"Índice {index_name} criado na tabela {nm_tabela} (não-único).")

    conn.commit()

finally:
    print(f'{fc.data_hora()} Finalizando criação de índices...')
    conn.close()

    def importar_csv_bulk(ambiente:str, caminho_csv:str, nome_tabela:str, separador='|', first_row_header=True):
"""
Importa dados CSV para SQL Server usando BULK INSERT nativo.

Esta função utiliza o comando BULK INSERT do SQL Server para importação
otimizada de grandes volumes de dados, oferecendo performance superior
ao INSERT tradicional.

Args:
    ambiente (str): Ambiente do banco ('PRODUCAO' ou 'HOMOLOGACAO')
    caminho_csv (str): Caminho absoluto para o arquivo CSV
    nome_tabela (str): Nome da tabela de destino
    separador (str, optional): Separador de campos do CSV. Defaults to '|'.
    first_row_header (bool, optional): Se primeira linha é cabeçalho. Defaults to True.

Examples:
    >>> importar_csv_bulk(
    ...     'PRODUCAO', 
    ...     'C:\\dados\\arquivo.csv', 
    ...     'innoveo_kovr_ap'
    ... )

Note:
    - Utiliza BULK INSERT com TABLOCK para máxima performance
    - FIELDTERMINATOR e ROWTERMINATOR configuráveis
    - FIRSTROW = 2 quando first_row_header=True (pula cabeçalho)
    - Caminho deve ser acessível pelo servidor SQL Server
    - Escapes de barras invertidas tratados automaticamente
"""
# Conexão
con = abrir_conexao_sqlserver(ambiente)
try:
    cur = con.cursor()
    header_option = "FIRSTROW = 2" if first_row_header else "FIRSTROW = 1"
    # Escapa corretamente as barras invertidas
    caminho_csv_escaped = caminho_csv.replace("\\", "\\\\")
    comando_bulk = f"""
        BULK INSERT {nome_tabela}
        FROM '{caminho_csv_escaped}'
        WITH (
            FIELDTERMINATOR = '{separador}',
            ROWTERMINATOR = '\\n',
            {header_option},
            TABLOCK
        )
    """
    cur.execute(comando_bulk)
    con.commit()
finally:
    con.close()

    def codificar_text_para_bytes(value):
"""
Converte UUID em vários formatos para Binary subtype 4 do MongoDB.

Esta função é essencial para interação com MongoDB, convertendo UUIDs
em formato binário compatível com as collections do banco. Aceita
múltiplos formatos de entrada e trata valores vazios adequadamente.

Args:
    value: UUID em formato uuid.UUID, bytes(16), string com/sem hífens,
           ou valores vazios (None, '')

Returns:
    bson.Binary or None: UUID em formato binário MongoDB ou None para valores vazios

Raises:
    ValueError: Para valores inválidos de UUID (bytes com tamanho incorreto, 
               strings malformadas)

    - Ignora valores vazios retornando None
    - Suporta UUIDs com e sem hífens
    - Essencial para queries MongoDB com UUIDs

Converte UUID em vários formatos para Binary subtype 4 do Mongo.
Aceita: uuid.UUID, bytes(16), string com/sem hífens.
Ignora valores vazios (None, '') retornando None.
"""

    return None

if isinstance(value, uuid.UUID):
    return Binary(value.bytes, subtype=UUID_SUBTYPE)

if isinstance(value, bytes):
    if len(value) == 16:
        return Binary(value, subtype=UUID_SUBTYPE)
    else:
        raise ValueError(f"Bytes inválidos para UUID: {value}")

if isinstance(value, str):
    clean_val = value.strip().lower()
    if clean_val == '':
        return None
    try:
        if '-' in clean_val:
            return Binary(uuid.UUID(clean_val).bytes, subtype=UUID_SUBTYPE)
        else:
            return Binary(uuid.UUID(hex=clean_val).bytes, subtype=UUID_SUBTYPE)
    except Exception as e:
        raise ValueError(f"String inválida para UUID: {value} ({e})")

raise TypeError(f"Tipo não suportado para UUID: {type(value)}")

    def converter_coluna_para_numerico(df, coluna):
"""
Converte coluna de texto com valores monetários brasileiros para numéricos.

Esta função trata formatação brasileira de números, removendo símbolos
monetários, espaços e convertendo vírgulas decimais para pontos.
Valores inválidos são convertidos para NaN automaticamente.

Args:
    df (pd.DataFrame): DataFrame contendo a coluna a ser convertida
    coluna (str): Nome da coluna para conversão

Returns:
    pd.Series: Série convertida para valores numéricos (float)

Examples:
    >>> import pandas as pd
    >>> df = pd.DataFrame({
    ...     'valores': ['R$ 1.234,56', '2.500,00', '1,50', 'N/A']
    ... })
    >>> df['valores_num'] = converter_coluna_para_numerico(df, 'valores')
    >>> print(df['valores_num'])
    0    1234.56
    1    2500.00
    2       1.50
    3        NaN

    >>> # Valores com separação de milhares
    >>> df2 = pd.DataFrame({
    ...     'preco': ['R$ 15.000,99', '1.000.000,50']
    ... })
    >>> converter_coluna_para_numerico(df2, 'preco')
    0      15000.99
    1    1000000.50

Note:
    - Remove R$, espaços e caracteres não-numéricos
    - Converte vírgula decimal para ponto
    - Preserva separação de milhares com ponto
    - Valores inválidos viram NaN (pd.to_numeric errors='coerce')
    - Ideal para processar dados financeiros brasileiros
"""
"""
Converte uma coluna de texto com vírgulas (e opcionalmente separadores de milhar)
para valores numéricos (float). Valores inválidos são convertidos para NaN.

Parâmetros:
    df (pd.DataFrame): O DataFrame contendo a coluna.
    coluna (str): O nome da coluna a ser convertida.

Retorna:
    pd.Series: A coluna convertida para numérico.
"""
return (
    df[coluna]
    .astype(str)
    .str.replace(r'[^\d,.-]', '', regex=True)  # remove R$, espaços, etc
    #.str.replace('.', '', regex=False)         # remove separador de milhar
    .str.replace(',', '.', regex=False)        # converte decimal vírgula para ponto
    .pipe(pd.to_numeric, errors='coerce')      # converte para número, inválidos viram NaN
)

    def formatar_datas_coluna_rapido(df, coluna, nova_coluna):
"""
Detecta e converte automaticamente múltiplos formatos de data para datetime.

Esta função utiliza uma lista de formatos comuns para detectar e converter
datas automaticamente, testando cada formato até encontrar uma correspondência
válida. É ideal para processar dados com formatos de data inconsistentes.

Args:
    df (pd.DataFrame): DataFrame contendo a coluna de datas
    coluna (str): Nome da coluna com datas em formato string
    nova_coluna (str): Nome da nova coluna que receberá as datas convertidas

Returns:
    pd.DataFrame: DataFrame original com nova coluna de datas convertidas

Examples:
    >>> import pandas as pd
    >>> df = pd.DataFrame({
    ...     'data_str': ['2025.02.01', '01/02/2025', '2025-02-01', '01-02-2025']
    ... })
    >>> df = formatar_datas_coluna_rapido(df, 'data_str', 'data_convertida')
    >>> print(df['data_convertida'])
    0   2025-02-01
    1   2025-02-01
    2   2025-02-01
    3   2025-02-01

    >>> # Com timestamps
    >>> df2 = pd.DataFrame({
    ...     'timestamp': ['2024.06.25 05:02:46']
    ... })
    >>> formatar_datas_coluna_rapido(df2, 'timestamp', 'datetime')

Note:
    - Formatos suportados:
      - AAAA.MM.DD 
      - AAAA-MM-DD
      - DD/MM/AAAA
      - MM/DD/AA
      - DD-MM-AAAA
      - AAAA/MM/DD
      - DD.MM.AAAA
      - AAAA.MM.DD HH:MM:SS
    - Processamento otimizado: para na primeira correspondência
    - Valores inválidos permanecem como NaT (Not a Time)
    - Performance otimizada para grandes volumes
"""
formatos = [
    "%Y.%m.%d",       # 2025.02.01
    "%Y-%m-%d",       # 2025-02-01
    "%d/%m/%Y",       # 01/02/2025
    "%m/%d/%y",       # 02/01/2025
    "%d-%m-%Y",       # 01-02-2025
    "%Y/%m/%d",       # 2025/02/01
    "%d.%m.%Y",       # 01.02.2025
    "%Y.%m.%d %H:%M:%S"        #2024.06.25 05:02:46
]

resultado = pd.Series(pd.NaT, index=df.index)

for fmt in formatos:
    mask = resultado.isna()
    if not mask.any():
        break
    resultado[mask] = pd.to_datetime(df.loc[mask, coluna], format=fmt, errors='coerce')

df[nova_coluna] = resultado
return df

    def mover_arquivos_diarios(path_arquivos):
"""
Move arquivos CSV do diretório STAGE para o diretório principal após processamento.

Esta função realiza a limpeza da pasta principal e depois move todos
os arquivos CSV processados da pasta STAGE para a pasta principal,
finalizando o ciclo de processamento.

Args:
    path_arquivos (str): Caminho base contendo as pastas principal e STAGE

Examples:
    >>> mover_arquivos_diarios('E:\\ARQUIVOS_MONGO\\kovr_ap')
    2025-09-19 14h 30m 45s Limpando pasta principal
    2025-09-19 14h 30m 46s Movendo 5 arquivos

    # Antes:
    # E:\ARQUIVOS_MONGO\kovr_ap\STAGE\2025-01-01.csv
    # E:\ARQUIVOS_MONGO\kovr_ap\STAGE\2025-01-02.csv

    # Depois:
    # E:\ARQUIVOS_MONGO\kovr_ap\2025-01-01.csv
    # E:\ARQUIVOS_MONGO\kovr_ap\2025-01-02.csv

Note:
    - ETAPA 1: Remove todos os .csv da pasta principal
    - ETAPA 2: Move todos os .csv da STAGE para principal
    - Arquivos são movidos (não copiados) - STAGE fica vazia
    - Logs informativos com contagem de arquivos
    - Operação segura - verifica existência antes de remover
"""
lista_arquivos = glob(os.path.join(path_arquivos,'*.csv'))
print(f' {data_hora()} Limpando pasta principal')
for arq in lista_arquivos:
    if os.path.exists(arq):
        os.remove(arq)


path_stage = os.path.join(path_arquivos,'STAGE')
lista_arquivos_stage = glob(os.path.join(path_stage,'*.csv'))

print(f' {data_hora()} Movendo {len(lista_arquivos_stage)} arquivos')
for arq in lista_arquivos_stage:
    shutil.move(arq,path_arquivos)
